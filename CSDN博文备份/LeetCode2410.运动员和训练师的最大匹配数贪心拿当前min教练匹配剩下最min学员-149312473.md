<h2><a id="LetMeFly2410minmin_1"></a>【LetMeFly】2410.运动员和训练师的最大匹配数：贪心（拿当前min教练匹配剩下最min学员）</h2> <br><p>力扣题目链接：<a href="https://leetcode.cn/problems/maximum-matching-of-players-with-trainers/" rel="nofollow">https://leetcode.cn/problems/maximum-matching-of-players-with-trainers/</a></p> <br><p>给你一个下标从 <strong>0</strong> 开始的整数数组 <code>players</code> ，其中 <code>players[i]</code> 表示第 <code>i</code> 名运动员的 <strong>能力</strong> 值，同时给你一个下标从 <strong>0</strong> 开始的整数数组 <code>trainers</code> ，其中 <code>trainers[j]</code> 表示第 <code>j</code> 名训练师的 <strong>训练能力值</strong> 。</p> <br><p>如果第 <code>i</code> 名运动员的能力值 <strong>小于等于</strong> 第 <code>j</code> 名训练师的能力值，那么第 <code>i</code> 名运动员可以 <strong>匹配</strong> 第 <code>j</code> 名训练师。除此以外，每名运动员至多可以匹配一位训练师，每位训练师最多可以匹配一位运动员。</p> <br><p>请你返回满足上述要求 <code>players</code> 和 <code>trainers</code> 的 <strong>最大</strong> 匹配数。</p> <br><p> </p> <br><p><strong>示例 1：</strong></p> <br><pre><strong>输入：</strong>players = [4,7,9], trainers = [8,2,5,8]<br><b>输出：</b>2<br><b>解释：</b><br>得到两个匹配的一种方案是：<br>- players[0] 与 trainers[0] 匹配，因为 4 <= 8 。<br>- players[1] 与 trainers[3] 匹配，因为 7 <= 8 。<br>可以证明 2 是可以形成的最大匹配数。<br></pre> <br><p><strong>示例 2：</strong></p> <br><pre><b>输入：</b>players = [1,1,1], trainers = [10]<br><b>输出：</b>1<br><b>解释：</b><br>训练师可以匹配所有 3 个运动员<br>每个运动员至多只能匹配一个训练师，所以最大答案是 1 。<br></pre> <br><p> </p> <br><p><strong>提示：</strong></p> <br><ul><li><code>1 <= players.length, trainers.length <= 10<sup>5</sup></code></li><li><code>1 <= players[i], trainers[j] <= 10<sup>9</sup></code></li></ul> <br><h3><a <p>当前最low教练能教小菜，也能教小强，教练教小菜是不是浪费了？不浪费，这个教练都能教小菜了，下一个教练更能教小菜。我们只关注被教的学员数量而不关注所教的学员水平。</p> <br> <p>为何不让小菜选一个超强教练？很显然，因为浪费。超强教练都拿来教小菜了，后面的学员小强就不一定能有教练教了。</p> <br></blockquote> <br><ul><li>时间复杂度<span
