<h2><a id="LetMeFly2266_1"></a>【LetMeFly】2266.统计打字方案数：排列组合</h2> <br><p>力扣题目链接：<a href="https://leetcode.cn/problems/count-number-of-texts/" rel="nofollow">https://leetcode.cn/problems/count-number-of-texts/</a></p> <br><p>Alice 在给 Bob 用手机打字。数字到字母的 <strong>对应</strong> 如下图所示。</p> <br><p><img alt="" src="https://i-blog.csdnimg.cn/img_convert/8dc91cb2c4d87af797b04ea5fd7d59a4.png" /></p> <br><p>为了 <strong>打出</strong> 一个字母，Alice 需要 <strong>按</strong> 对应字母 <code>i</code> 次，<code>i</code> 是该字母在这个按键上所处的位置。</p> <br><ul><li>比方说，为了按出字母 <code>'s'</code> ，Alice 需要按 <code>'7'</code> 四次。类似的， Alice 需要按 <code>'5'</code> 两次得到字母  <code>'k'</code> 。</li><li>注意，数字 <code>'0'</code> 和 <code>'1'</code> 不映射到任何字母，所以 Alice <strong>不</strong> 使用它们。</li></ul> <br><p>但是，由于传输的错误，Bob 没有收到 Alice 打字的字母信息，反而收到了 <strong>按键的字符串信息</strong> 。</p> <br><ul><li>比方说，Alice 发出的信息为 <code>"bob"</code> ，Bob 将收到字符串 <code>"2266622"</code> 。</li></ul> <br><p>给你一个字符串 <code>pressedKeys</code> ，表示 Bob 收到的字符串，请你返回 Alice <strong>总共可能发出多少种文字信息</strong> 。</p> <br><p>由于答案可能很大，将它对 <code>10<sup>9</sup> + 7</code> <strong>取余</strong> 后返回。</p> <br><p> </p> <br><p><strong>示例 1：</strong></p> <br><pre><b>输入：</b>pressedKeys = "22233"<br><b>输出：</b>8<br><strong>解释：</strong><br>Alice 可能发出的文字信息包括：<br>"aaadd", "abdd", "badd", "cdd", "aaae", "abe", "bae" 和 "ce" 。<br>由于总共有 8 种可能的信息，所以我们返回 8 。<br></pre> <br><p><strong>示例 2：</strong></p> <br><pre><b>输入：</b>pressedKeys = "222222222222222222222222222222222222"<br><b>输出：</b>82876089<br><strong>解释：</strong><br>总共有 2082876103 种 Alice 可能发出的文字信息。<br>由于我们需要将答案对 10<sup>9</sup> + 7 取余，所以我们返回 2082876103 % (10<sup>9</sup> + 7) = 82876089 。<br></pre> <br><p> </p> <br><p><strong>提示：</strong></p> <br><ul><li><code>1 <= pressedKeys.length <= 10<sup>5</sup></code></li><li><code>pressedKeys</code> 只包含数字 <code>'2'</code> 到 <code>'9'</code> 。</li></ul> <br><h3><a id="_60"></a>解题方法：排列组合</h3> <br><p>按键<code>2</code>可以按<code>1/2/3</code>次，那么<code>n</code>次连续的按键<code>2</code>共有多少种拼写可能？</p> <br><blockquote> <br> <p>使用数组<code>dp3</code>，其中<code>dp3[n]</code>代表连续按<code>2</code>键<code>n</code>次有多少种拼写方案。我们考虑最后一个按出来的字母：</p> <br> <ol><li>第<code>n</code>次按<code>2</code>可以在按了<code>n - 1</code>次的基础上再按<code>1</code>次（得到<code>a</code>）</li><li>第<code>n</code>次按<code>2</code>可以在按了<code>n - 2</code>次的基础上连续按<code>2</code>次（得到<code>b</code>）</li><li>第<code>n</code>次按<code>2</code>可以在按了<code>n - 3</code>次的基础上连续按<code>3</code>次（得到<code>c</code>）</li></ol> <br> <p>因此<code>dp3[n] = (dp3[n - 1] + dp3[n - 2] + dp3[n - 3]) % mod</code></p> <br></blockquote> <br><p>和按键<code>2</code>一样，按键<code>3/4/5/6/8</code>同样适用于<code>dp3</code>数组；而按键<code>7</code>和按键<code>9</code>最多可以连按<code>4</code>次，因此可以计算<code>dp4</code>数组：</p> <br><blockquote> <br> <p><code>dp4[n] = (dp4[n - 1] + dp4[n - 2] + dp4[n - 3] + dp4[n - 4]) % mod</code></p> <br></blockquote> <br><p>给你一个按键字符串如<code>22233</code>，我们可以通过一次遍历很轻松地得到“连按了<code>3</code>次<code>2</code>，又连按了<code>2</code>次<code>3</code>”这一信息。</p> <br><p>连按<code>3</code>次<code>2</code>可以有<code>dp3[3]</code>种方案，连按<code>2</code>次<code>3</code>可以有<code>dp3[2]</code>种方案，因此依据乘法原理，总计有<code>dp3[3] * dp3[2]</code>种方案。</p> <br><ul><li>时间复杂度<span
