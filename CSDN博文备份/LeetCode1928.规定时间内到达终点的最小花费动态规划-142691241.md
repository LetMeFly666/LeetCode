<h2><a id="LetMeFly1928_1"></a>【LetMeFly】1928.规定时间内到达终点的最小花费：动态规划</h2> <br><p>力扣题目链接：<a href="https://leetcode.cn/problems/minimum-cost-to-reach-destination-in-time/" rel="nofollow">https://leetcode.cn/problems/minimum-cost-to-reach-destination-in-time/</a></p> <br><p>一个国家有 <code>n</code> 个城市，城市编号为 <code>0</code> 到 <code>n - 1</code> ，题目保证 <strong>所有城市</strong> 都由双向道路 <b>连接在一起</b> 。道路由二维整数数组 <code>edges</code> 表示，其中 <code>edges[i] = [x<sub>i</sub>, y<sub>i</sub>, time<sub>i</sub>]</code> 表示城市 <code>x<sub>i</sub></code> 和 <code>y<sub>i</sub></code> 之间有一条双向道路，耗费时间为 <code>time<sub>i</sub></code> 分钟。两个城市之间可能会有多条耗费时间不同的道路，但是不会有道路两头连接着同一座城市。</p> <br><p>每次经过一个城市时，你需要付通行费。通行费用一个长度为 <code>n</code> 且下标从 <strong>0</strong> 开始的整数数组 <code>passingFees</code> 表示，其中 <code>passingFees[j]</code> 是你经过城市 <code>j</code> 需要支付的费用。</p> <br><p>一开始，你在城市 <code>0</code> ，你想要在 <code>maxTime</code> <strong>分钟以内</strong> （包含 <code>maxTime</code> 分钟）到达城市 <code>n - 1</code> 。旅行的 <strong>费用</strong> 为你经过的所有城市 <strong>通行费之和</strong> （<strong>包括</strong> 起点和终点城市的通行费）。</p> <br><p>给你 <code>maxTime</code>，<code>edges</code> 和 <code>passingFees</code> ，请你返回完成旅行的 <strong>最小费用</strong> ，如果无法在 <code>maxTime</code> 分钟以内完成旅行，请你返回 <code>-1</code> 。</p> <br><p> </p> src="https://img-home.csdnimg.cn/images/20230724024159.png?origin_url=https%3A%2F%2Fassets.leetcode.com%2Fuploads%2F2021%2F06%2F04%2Fleetgraph1-1.png&pos_id=img-b6IvYEOY-1727933171724%29" /></p> <br><pre><b>输入：</b>maxTime = 30, edges = [[0,1,10],[1,2,10],[2,5,10],[0,3,1],[3,4,10],[4,5,15]], passingFees = [5,1,2,20,20,3]<br><b>输出：</b>11<br><b>解释：</b>最优路径为 0 -> 1 -> 2 -> 5 ，总共需要耗费 30 分钟，需要支付 11 的通行费。<br></pre> <br><p><strong>示例 2：</strong></p> <br><p><strong><img alt="" src="https://img-blog.csdnimg.cn/img_convert/50c013cc302796b1a6f82fc979bb8e7d.png" /></strong></p> <br><pre><b>输入：</b>maxTime = 29, edges = [[0,1,10],[1,2,10],[2,5,10],[0,3,1],[3,4,10],[4,5,15]], passingFees = [5,1,2,20,20,3]<br><b>输出：</b>48<br><b>解释：</b>最优路径为 0 -> 3 -> 4 -> 5 ，总共需要耗费 26 分钟，需要支付 48 的通行费。<br>你不能选择路径 0 -> 1 -> 2 -> 5 ，因为这条路径耗费的时间太长。<br></pre> <br><p><strong>示例 3：</strong></p> <br><pre><b>输入：</b>maxTime = 25, edges = [[0,1,10],[1,2,10],[2,5,10],[0,3,1],[3,4,10],[4,5,15]], passingFees = [5,1,2,20,20,3]<br><b>输出：</b>-1<br><b>解释：</b>无法在 25 分钟以内从城市 0 到达城市 5 。<br></pre> <br><p> </p> <br><p><strong>提示：</strong></p> <br><ul><li><code>1 <= maxTime <= 1000</code></li><li><code>n == passingFees.length</code></li><li><code>2 <= n <= 1000</code></li><li><code>n - 1 <= edges.length <= 1000</code></li><li><code>0 <= x<sub>i</sub>, y<sub>i</sub> <= n - 1</code></li><li><code>1 <= time<sub>i</sub> <= 1000</code></li><li><code>1 <= passingFees[j] <= 1000</code> </li><li>图中两个节点之间可能有多条路径。</li><li>图中不含有自环。</li></ul> <br><h3><a id="_62"></a>解题方法：动态规划</h3> <br><p>使用<code>dp[t][i]</code>表示在时间<code>t</code>到达城市<code>i</code>所需最小花费。初始值除了<code>dp[0][0] = passingFees[0]</code>外其余<code>dp[t][i]</code>全为“无穷大”<code>10^7</code>。</p> <br><p>对于当前时间<code>t</code>，枚举每一条边（假设这条边连接<code>i</code>和<code>j</code>而花费是<code>cost</code>），若<code>cost <= t</code>，则能更新<code>dp[t][j] = min(dp[t][j], dp[t - cost][i] + passingFees)</code>，<code>dp[t][i]</code>同理。</p> <br><p>最终，对于所有的<code>t</code>，<code>dp[t][n - 1]</code>最小的那个即为最小耗时（若为“无穷大”则返回<code>-1</code>）。</p> <br><ul><li>时间复杂度<span class="katex--inline"><span
