<h2><a id="LetMeFly63_II___1"></a>【LetMeFly】63.不同路径 II：动态规划 - 原地使用地图数组，几乎无额外空间开销</h2> <br><p>力扣题目链接：<a href="https://leetcode.cn/problems/unique-paths-ii/" rel="nofollow">https://leetcode.cn/problems/unique-paths-ii/</a></p> <br><p>给定一个 <code>m x n</code> 的整数数组 <code>grid</code>。一个机器人初始位于 <strong>左上角</strong>（即 <code>grid[0][0]</code>）。机器人尝试移动到 <strong>右下角</strong>（即 <code>grid[m - 1][n - 1]</code>）。机器人每次只能向下或者向右移动一步。</p> <br><p>网格中的障碍物和空位置分别用 <code>1</code> 和 <code>0</code> 来表示。机器人的移动路径中不能包含 <strong>任何</strong> 有障碍物的方格。</p> <br><p>返回机器人能够到达右下角的不同路径数量。</p> <br><p>测试用例保证答案小于等于 <code>2 * 10<sup>9</sup></code>。</p> <br><p> </p> <br><p><strong>示例 1：</strong></p> <br><img alt="" src="https://i-blog.csdnimg.cn/img_convert/df396fa1cd39b661def6233b48814a2c.jpeg" /> <br><pre><strong>输入：</strong>obstacleGrid = [[0,0,0],[0,1,0],[0,0,0]]<br><strong>输出：</strong>2<br><strong>解释：</strong>3x3 网格的正中间有一个障碍物。<br>从左上角到右下角一共有 <code>2</code> 条不同的路径：<br>1. 向右 -> 向右 -> 向下 -> 向下<br>2. 向下 -> 向下 -> 向右 -> 向右<br></pre> <br><p><strong>示例 2：</strong></p> <br><img alt="" src="https://i-blog.csdnimg.cn/img_convert/4d3ba7903d75e2d68a7152a1a626b247.jpeg" /> <br><pre><strong>输入：</strong>obstacleGrid = [[0,1],[0,0]]<br><strong>输出：</strong>1<br></pre> <br><p> </p> <br><p><strong>提示：</strong></p> <br><ul><li><code>m == obstacleGrid.length</code></li><li><code>n == obstacleGrid[i].length</code></li><li><code>1 <= m, n <= 100</code></li><li><code>obstacleGrid[i][j]</code> 为 <code>0</code> 或 <code>1</code></li></ul> <br><h3><a id="_46"></a>解题方法：动态规划</h3> <br><p>直接使用原来的obstacleGrid数组，令<code>obstacleGrid[i][j]</code>代表走到<code>(i, j)</code>为止的总方案数。</p> <br><p>因为是原地修改，所以就要求我们从左到右从上到下按顺序遍历。</p> <br><p>遍历到一个元素时：</p> <br><ul><li>如果这个元素为<code>1</code>，就说明这里是障碍，走这里的方案数为<code>0</code>；</li><li>否则，走这里的方案数为“由上面来”和“由左边来”的方案数之和（若不可由上而来则将“由上面来”的方案数记为1）。</li></ul> <br><p>特别的，对于起始位置<code>obstacleGrid[0][0]</code>：</p> <br><ul><li>若初始值为<code>1</code>说明不可从这里出发，总方案数为<code>0</code>；</li><li>若初始值为<code>0</code>说明可以从这里出发，令<code>obstacleGrid[0][0] = 1</code>。</li></ul> <br><h4><a id="_62"></a>时空复杂度分析</h4>
