<h2><a id="LetMeFly3439_I__k_1"></a>【LetMeFly】3439.重新安排会议得到最多空余时间 I：贪心(滑动窗口) - 连开k场会议</h2> <br><p>力扣题目链接：<a href="https://leetcode.cn/problems/reschedule-meetings-for-maximum-free-time-i/" rel="nofollow">https://leetcode.cn/problems/reschedule-meetings-for-maximum-free-time-i/</a></p> <br><p>给你一个整数 <code>eventTime</code> 表示一个活动的总时长，这个活动开始于 <code>t = 0</code> ，结束于 <code>t = eventTime</code> 。</p> <br><p>同时给你两个长度为 <code>n</code> 的整数数组 <code>startTime</code> 和 <code>endTime</code> 。它们表示这次活动中 <code>n</code> 个时间 <strong>没有重叠</strong> 的会议，其中第 <code>i</code> 个会议的时间为 <code>[startTime[i], endTime[i]]</code> 。</p> <br><p>你可以重新安排 <strong>至多</strong> <code>k</code> 个会议，安排的规则是将会议时间平移，且保持原来的 <strong>会议时长</strong> ，你的目的是移动会议后 <strong>最大化</strong> 相邻两个会议之间的 <strong>最长</strong> 连续空余时间。</p> <br><p>移动前后所有会议之间的 <strong>相对</strong> 顺序需要保持不变，而且会议时间也需要保持互不重叠。</p> <br><p>请你返回重新安排会议以后，可以得到的 <strong>最大</strong> 空余时间。</p> <br><p><b>注意</b>，会议 <strong>不能</strong> 安排到整个活动的时间以外。</p> <br><p> </p> <br><p><strong class="example">示例 1：</strong></p> <br><div class="example-block"> <br> <p><span class="example-io"><b>输入：</b>eventTime = 5, k = 1, startTime = [1,3], endTime = [2,5]</span></p> <br></div><br><p><span class="example-io"><b>输出：</b>2</span></p> <br><p><strong>解释：</strong></p> <br><p><img alt="" src="https://i-blog.csdnimg.cn/img_convert/8b9e4269152ff6234d1fa0b8fc2d0b48.png" /></p> <br><p>将 <code>[1, 2]</code> 的会议安排到 <code>[2, 3]</code> ，得到空余时间 <code>[0, 2]</code> 。</p> <br><p><strong class="example">示例 2：</strong></p> <br><div class="example-block"> <br> <p><span class="example-io"><b>输入：</b>eventTime = 10, k = 1, startTime = [0,2,9], endTime = [1,4,10]</span></p> <br></div><br><p><span class="example-io"><b>输出：</b>6</span></p> <br><p><strong>解释：</strong></p> <br><p><img alt="" src="https://i-blog.csdnimg.cn/img_convert/1c562e3b039867d879be4859810a072b.png" /></p> <br><p>将 <code>[2, 4]</code> 的会议安排到 <code>[1, 3]</code> ，得到空余时间 <code>[3, 9]</code> 。</p> <br><p><strong class="example">示例 3：</strong></p> <br><div class="example-block"> <br> <p><span class="example-io"><b>输入：</b>eventTime = 5, k = 2, startTime = [0,1,2,3,4], endTime = [1,2,3,4,5]</span></p> <br></div><br><p><span class="example-io"><b>输出：</b>0</span></p> <br><p><strong>解释：</strong></p> <br><p>活动中的所有时间都被会议安排满了。</p> <br><p> </p> <br><p><strong>提示：</strong></p> <br><ul><li><code>1 <= eventTime <= 10<sup>9</sup></code></li><li><code>n == startTime.length == endTime.length</code></li><li><code>2 <= n <= 10<sup>5</sup></code></li><li><code>1 <= k <= n</code></li><li><code>0 <= startTime[i] < endTime[i] <= eventTime</code></li><li><code>endTime[i] <= startTime[i + 1]</code> 其中 <code>i</code> 在范围 <code>[0, n - 2]</code> 之间。</li></ul> <br><h3><a id="_74"></a>解题方法：滑动窗口</h3> <br><h4><a id="_76"></a>解题思路</h4> <br><p>怎么尽可能出现“最大间隙”？当然是尽可能地把会放到一块开。</p> <br><blockquote> <br> <p>题外话：长痛不如短痛，一下连着把课上了然后连着放个长假挺好的。</p> <br></blockquote> <br><p>但是，我们最多调整k个会议的时间。怎么调？选连续的k个挤到一块呗（把当前选中的4个放到<span title="或0">上次会议的end</span>之后紧挨着）。</p> <br><p>每次把连续的<span <br><p>首先将前k场会议统一挪到0时刻开始紧挨着k场。我们把这k场视为“窗口”，那么下一个“窗口”就是这次窗口右移一位（加上下一场会议减去第一场会议）。</p> <br><p>使用一个变量cnt记录当前窗口中k场会议的总时间，根据（其实只需要加上→）窗口的上一场会议的结束时间就能得到这k场会议全部结束的时间了。</p> <br><h4><a id="_92"></a>时空复杂度分析</h4>
